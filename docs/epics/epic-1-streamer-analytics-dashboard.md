# Epic 1：實況主長期營運儀表板（Streamer Analytics Dashboard）

## 1. Epic 概述

### 1.1 背景

實況主目前多依賴 Twitch 內建後台或零散第三方工具，只能看到單場或短期數據，難以掌握「長期開台習慣、訂閱趨勢與觀眾行為」。本 Epic 專注提供一個集中且易讀的「長期營運儀表板」，協助實況主做營運與排程決策。

### 1.2 Epic 目標

- 提供實況主一個可檢視「指定期間」內開台表現的總覽。
- 讓實況主理解自己在「時間分布、頻率、訂閱變化」上的長期趨勢。
- 打造一個可持續擴充、體驗良好的 Dashboard 作為未來功能基礎。

---

## 2. 範圍（Scope）

### 2.1 In Scope

- 實況主登入與 Twitch 頻道綁定。
- 指定期間（如 7 / 30 / 90 天）的：
  - 開台總時數、總場數、平均單場長度。
  - 開台時間分布與頻率圖表（按日 / 週 / 時段）。
- 基礎訂閱數變化趨勢視覺化（受限於 Twitch API 能力）。
- 儀表板基本 UX：卡片、圖表、期間切換、顯示偏好設定。

### 2.2 Out of Scope（本 Epic 不做）

- 觀眾個人統計頁（會在另一個 Epic）。
- 進階營運建議（例如自動推薦最佳開台時段）。
- 跨平台整合（YouTube、Kick 等）。

---

## 3. 使用者價值（User Value）

- 實況主能以一個頁面快速掌握近期表現，而非到處對比截圖或報表。
- 有助於決定：
  - 是否調整開台時段。
  - 是否增加 / 減少開台頻率。
  - 活動或新嘗試對訂閱與開台習慣的影響。

---

## 4. User Stories（Epic 內包含的 Stories）

> 下列 Stories 對應 PRD 內的 Streamer Analytics
> Dashboard 範圍，實際實作時可各自建立獨立 Story 文件。

### Story 1.1 – 實況主登入與頻道綁定

- **As a** 實況主
- **I want** 使用我的 Twitch 帳號登入並綁定頻道
- **So that** 系統能自動為我的頻道收集與顯示統計數據

**重點**：

- Twitch OAuth 登入流程。
- 建立 / 更新實況主帳號紀錄與授權資訊。
- 綁定並顯示主要頻道名稱、頭像與連結。

---

### Story 1.2 – 指定期間開台統計總覽

- **As a** 實況主
- **I want** 查看特定期間的開台統計總覽
- **So that** 我能快速掌握自己近期的開台量與活躍程度

**重點**：

- 可切換時間區間（預設最近 30 天，支援 7 / 90 天）。
- 顯示總開台時數、總場數、平均單場長度等 Summary Cards。

---

### Story 1.3 – 開台時間與頻率圖表

- **As a** 實況主
- **I want** 以圖表方式查看開台時間分布與頻率
- **So that** 我能調整開台時段以優化表現

**重點**：

- 折線 / 柱狀圖顯示每日或每週開台時數。
- 簡化 Heatmap 或時段分布圖顯示常見開台時段。

---

### Story 1.4 – 訂閱數變化趨勢（簡版）

- **As a** 實況主
- **I want** 查看指定期間內訂閱數的變化趨勢
- **So that** 我能判斷近期活動或內容策略是否影響訂閱

**重點**：

- 視 Twitch API 能力顯示每日 / 每週訂閱總數或淨變化。
- 若資料有限需在 UI 清楚標示指標的限制。

---

### Story 1.5 – 儀表板 UX 與顯示偏好設定

- **As a** 實況主
- **I want** 能簡單調整儀表板顯示內容
- **So that** 我能專注在我最在意的指標

**重點**：

- 可隱藏 / 顯示特定卡片或圖表。
- 儲存顯示偏好（先可用 LocalStorage，未來可改為後端儲存）。

---

## 5. 非功能與 UX 要點（本 Epic 聚焦部分）

- 前端技術棧：React + TypeScript + Tailwind CSS。
- 儀表板主要互動（切換時間區間、切換圖表 Tab）在一般網路環境下應於 1 秒內回應。
- 圖表與卡片需：
  - 有清楚標題與單位（小時 / 次數）。
  - 滑鼠 hover 時提供簡潔 tooltip。
  - 提供簡要文字敘述，協助可訪問性。

---

## 6. 骨幹成功指標（Epic Level）

- 至少有 N 位實況主成功登入並綁定頻道，且實際開啟儀表板頁面 ≥ 2 次。
- 實況主訪談中，有 ≥ X% 回覆「Dashboard 有幫助我理解近期或長期表現」。
- 儀表板頁面主要互動（切換期間、切 Tab）平均回應時間 ≤ 1 秒。

（N / X 具體數字可依專案實際規模再行填寫）
