# 專案簡報：Twitch 實況監控與統計平台（暫名）

## 1. 專案概要（Project Overview）

### 1.1 一句話描述

提供實況主與重度觀眾一個專門用來「長期追蹤與分析 Twitch 行為數據」的網站，聚焦開台行為、訂閱與互動表現。

### 1.2 背景與問題點

- Twitch 本身雖然提供部分後台數據與第三方工具，但多數工具偏向「單一面向」或「短期觀看」，例如只看當下同時在線、近期幾場直播表現。
- 實況主若想了解「長期開台習慣、訂閱趨勢、鐵粉互動行為」，往往需要手動整理、截圖或下載報表，再自己做分析，成本高且不連續。
- 重度觀眾（或管理員 / 社群營運）若想了解「自己在某台、某段期間的觀看投入與互動情況」，目前缺乏一個好用的、以「觀眾視角」為主的統計工具。

### 1.3 解決方案概念（Solution Idea）

- 建立一個 Web 平台，透過 Twitch API 等資料來源，持續監控與儲存指定實況主與觀眾的相關數據。
- 為實況主提供：開台時數、開台頻率、平均 / 峰值人數、訂閱數變化、關鍵時段表現等圖表與趨勢分析（初期先聚焦開台與訂閱趨勢）。
- 為觀眾提供：在特定實況主頻道的觀看時數、互動次數（留言數為主），以及個人「追台足跡」概覽。

### 1.4 目標用戶（Target Users）

1. 實況主（Streamer）
2. 重度觀眾 / 頻道經理 / 社群營運

### 1.5 核心價值主張（Value Proposition）

- 比 Twitch 內建後台與一般第三方工具更聚焦「長期行為與關係」，而不是只看單場或短期表現。
- 同時服務「實況主」與「觀眾」，形成一個以數據為核心的互動生態資訊面板。
- 以「自動、持續、可視化」為核心，不需要使用者自己 export / excel 分析。

## 2. 目標、範圍與成功指標（Goals, Scope & Success Metrics）

### 2.1 專案目標（Goals）

1. 為實況主提供「長期可視化」的營運儀表板。
2. 為觀眾提供「個人觀影足跡與互動紀錄」。
3. 為未來資料分析與進階功能打基礎。

### 2.2 初步功能範圍（MVP Scope）

#### 實況主側（Streamer View – MVP）

- Twitch OAuth 登入與頻道綁定（實作細節待後續架構設計）。
- 儀表板：
  - 指定期間內的總開台時數 / 場數 / 平均單場長度。
  - 以日 / 週 / 月為單位的開台時數圖表。
  - 簡化版訂閱數變化趨勢（視 API 能力）。

#### 觀眾側（Viewer View – MVP）

- 觀眾在特定實況主頻道的：
  - 指定期間累積觀看時數。
  - 指定期間累積留言數。

#### 共用基礎

- 以「鎖定一個實況主」為核心視角，在其下切換「實況主總覽 / 單一觀眾」。
- 基礎身份驗證與權限控制（實況主只看自己頻道聚合資料，觀眾只看與自己相關的統計）。

### 2.3 初步成功指標（Success Metrics – Draft）

- 使用層面：
  - 上線後 3 個月內，有 N 位實況主綁定頻道。
  - 綁定實況主中至少 50% 每週登入一次查看儀表板。
- 價值感受層面（主觀）：
  - 實況主認為「有助於了解長期表現」的比例 ≥ X%。
  - 觀眾認為「能更清楚看到自己的支持度」的比例 ≥ Y%。

（N / X / Y 數值待後續確認）

## 3. 風險、限制與技術假設（Risks, Constraints & Assumptions）

### 3.1 主要風險（Key Risks）

1. Twitch API 限制與政策變動風險。
2. 資料隱私與授權風險（特別是觀眾端）。
3. 長期資料儲存與成本風險。
4. 實況主與觀眾的採用率風險。

### 3.2 主要限制（Constraints）

1. 嚴格受限於 Twitch 官方 API 能提供的資料。
2. 個人 / 小團隊開發資源有限。
3. 基礎設施與預算限制，初版架構需保持簡單。

### 3.3 技術假設（Technical Assumptions）

1. 資料來源以 Twitch 官方 API 為主，不使用違反條款的抓取方式。
2. 採 Web 前端 + 後端服務 + 排程 Job 架構：
   - 後端定時抓取數據存入 DB。
   - 前端提供 dashboard 與查詢。
3. 使用關聯式或時序型資料庫儲存開台與互動統計（以彙總資料為主）。
4. 使用 Twitch OAuth 進行登入，系統不自管密碼。

## 4. 開發階段規劃（Roadmap）

### 4.1 V0 – 技術驗證版（Prototype）

- Twitch API 串接與資料抓取 → DB → 簡易前端顯示。
- 單一實況主 Demo 儀表板（最近 7 / 30 天的開台時數與場數）。

### 4.2 V1 – 實況主版 MVP

- 實況主 Twitch OAuth 登入與頻道綁定。
- 實況主儀表板：
  - 指定期間開台時數、場數、平均長度、開台時間分佈。
  - 簡化訂閱數走勢（若可行）。
- 基本設定（啟用 / 停用某些統計）。

### 4.3 V2 – 觀眾版 + 進階分析雛形

- 觀眾 OAuth 登入與個人儀表板：
  - 在特定實況主頻道的觀看時數與留言數統計。
- 觀眾足跡概覽（常見觀看時段）。
- 實況主端新增「觀眾概況」彙總統計。

---

## 5. 後續可考慮的延伸方向（可選）

- 更細的互動指標（表情、禮物、訂閱續訂行為等，視 API 能力）。
- 鐵粉分層與標籤（例如 Top X% 觀看時數 / 留言數）。
- 活動前後對比報表（例如周年台、合作台等）。
- 推薦開台時段 / 內容類型的分析建議。
